## The idea

Each lab is its **own folder** with a `README.md` and – where needed – the
matching example and configuration files.
The labs build on each other: from your first repository on github.com to
**automated workflows with GitHub Actions**.

Every lab follows the same structure:

1. **What?** – What will you learn in this lab?
2. **Why?** – Why do you need this in practice?
3. **Prerequisites** – What needs to be in place beforehand?
4. **Hands-on** – Step-by-step commands.
5. **Observations** – What should you notice?
6. **Cleanup** – How do you tidy up afterwards?

## Prerequisites for the whole series

- **GitHub account** (free at https://github.com/)
- **Git for Windows** installed (from Lab 2 onwards) – https://git-scm.com/download/win
- **VS Code** (recommended) with built-in Git integration
- **PowerShell** (Windows) or **Bash** (macOS/Linux)
- Internet access

## Recommended order

Please work through the labs **in order** – each lab builds on the concepts
from the previous one.
